# p24 Factorized Certificate Specification

This is the current shortest explicit certificate surface for the mixed
Hermitian Schur target at

```text
p = 10^24 + 7.
```

It does not yet prove the required p-units.  It specifies exactly which
p-units would constitute a strict certificate and how they imply the
sub-sqrt decomposed route.

## Degree Accounting

The relevant CM order has

```text
h = 205880396014 = m*n
m = 66254 = 2*157*211
n = 3107441
sqrt(p) = 1000000000000
```

The decomposed target would use degrees

```text
quotient degree = m = 66254
recovery degree = n = 3107441
formal degree sum = m+n = 3173695
```

Thus the formal degree ratio is

```text
(m+n)/sqrt(p) ~= 3.173695e-6.
```

So the asymptotic win is enormous if the embedded quotient/recovery
certificate can be made seedless.

## Mixed Schur Gate

The current finite obstruction has been reduced to the mixed centered
marginal rank theorem:

```text
rank_Fp C_{157,211} = 156.
```

Equivalently, with

```text
L = F_p(mu_157),  [L:F_p]=156,
R = F_p(mu_211),  [R:F_p]=35,
```

the six right relative traces

```text
a_j(lambda) = Tr_{E/R}(lambda*S_j),   j=1,...,6
```

separate every nonzero `lambda in L` even after deleting any one right orbit.
This delete-one theorem implies right support `>=2`, hence the centered rank
theorem.

There is also a weaker but sufficient centered-profile route.  Let

```text
G_s^0 in L,       s mod 211,
```

be the centered right profile from
`p24/hermitian_mixed_centered_right_profile_theorem.md`.  If the first `156`
profile values span `L`, then:

```text
rank_Fp C_{157,211} = 156.
```

The corresponding manifest is:

```text
p24/p24_centered_profile_manifest.py
```

It names four equivalent centered-profile p-unit surfaces:

```text
M_profile_leading =
  det((G_s^0)^(p^i))_{0<=s,i<156};

Gamma_profile_leading =
  det(Tr_{L/F_p}(G_s^0*G_t^0))_{0<=s,t<156}
  = M_profile_leading^2;

B_profile_leading =
  det(A_lead^T * J_inv * A_lead),
  with Gamma_profile_leading = 157^156 * B_profile_leading;

D_profile_leading =
  det(A_lead with row r=0 dropped),
  with B_profile_leading = 157 * D_profile_leading^2.
```

The last form is entirely over the base field: `A_lead` is the centered
left-coefficient matrix.  Thus `D_profile_leading != 0` is the simplest
centered-profile p-unit certificate.  This route proves the mixed rank
theorem but not the stronger delete-one right-support theorem.

A still simpler sufficient, not equivalent, base-field route is:

```text
Delta_C_leading =
  det(C(r,s))_{1 <= r <= 156, 1 <= s <= 156},
```

where

```text
C(r,s)=M(r,s)-M(r,0)-M(0,s)+M(0,0)
```

is the doubly-centered `156 x 210` marginal.  Nonvanishing of
`Delta_C_leading` directly implies `rank_Fp C_{157,211}=156`.  This is the
current cleanest visible-minor theorem candidate; see
`p24/centered_marginal_difference_minor_theorem.md`.

## Paired Factor Certificate

The explicit p24 orbit/factor manifest is generated by:

```text
p24/p24_factorized_certificate_manifest.py
```

It yields an optimal paired four-block certificate:

```text
deleted O1/O2 share B={O3,O4,O5,O6}, tails O2/O1
deleted O3/O4 share B={O1,O2,O5,O6}, tails O4/O3
deleted O5/O6 share B={O1,O2,O3,O4}, tails O6/O5
```

So the named finite certificate consists of:

```text
3 prefix p-units B_A, B_B, B_C,
6 quotient-tail p-units T_1,...,T_6.
```

The three prefix factors have rank target:

```text
B != 0  <=>  four full right blocks have rank 140.
```

Each tail factor has rank target:

```text
T_j != 0  <=>  first 16 coordinates of the paired tail block add rank 16
               modulo the corresponding four-block span.
```

Together:

```text
B_pair(j) != 0 and T_j != 0
  => a 156-coordinate leading Moore minor is nonzero after deleting O_j
  => delete-one separation for O_j.
```

Checking all six deleted orbits proves the mixed Schur theorem.

There is now also a symmetry-aligned optimal pairing.  Negation on right
frequencies sends

```text
O1 <-> O4,  O2 <-> O5,  O3 <-> O6.
```

Using these opposite pairs gives:

```text
deleted O1/O4 share B={O2,O3,O5,O6}, tails O4/O1
deleted O2/O5 share B={O1,O3,O4,O6}, tails O5/O2
deleted O3/O6 share B={O1,O2,O4,O5}, tails O6/O3
```

The finite certificate still has six deletion rows, but this pairing exposes a
finite algebra compression: in the full cyclotomic product algebra, inversion
intertwines the opposite tail windows once compatible Lang-coordinate windows
are used.  Thus the six tail p-units fall into three conjugate pairs.
The natural smallest-representative orbit order does not have this property,
so the opposite manifest uses rotated q-consecutive windows chosen in pairs,
with the `O4`, `O5`, and `O6` windows equal to the negatives of the chosen
`O1`, `O2`, and `O3` windows in Lang coordinates.  The raw DFT seed has an
extra left-negation shift because `-1 = p^78 mod 157`; after Lang
trivialization that shift becomes coordinatewise Frobenius and does not move
the coordinate index.  This convention is audited in:

```text
p24/opposite_conjugation_window_audit.py
```

The finite theorem is recorded in:

```text
p24/opposite_conjugation_tail_theorem.md
```

The right-unit action on the full product algebra compresses these classes one
step further.  Multiplication by `2 mod 211` cycles all six right Frobenius
factors and cycles the three opposite prefixes.  With equivariant Lang-basis
choices, this reduces the proof surface to:

```text
1 representative leading Moore p-unit L = B*T.
```

The unit-action theorem and audit are:

```text

```

The current trace-GCD version of this representative p-unit has its own

The manifest records four verifier surfaces: 211 pointwise determinants plus

## Verifier Shape

An equivariant finite verifier for a supplied arithmetic certificate should
check:

1. the six right Frobenius orbits modulo `211`;
2. the negation action `O1<->O4`, `O2<->O5`, `O3<->O6`;
3. the unit-2 action cycling `O1->O2->...->O6->O1`;
4. the representative row:

```text
deleted O4,
B_prefix={O2,O3,O5,O6},
T_tail=O1,
tail_window=first 16 Lang coordinates of O1;
```

The same data is emitted as JSON by:

```text
p24/p24_equivariant_manifest.py
```

5. the representative leading Moore product `L=B*T` is nonzero in `F_p`;
6. optionally, the factorization into `B` and `T` if the certificate is
   supplied in split form;
7. the product-algebra equivariance data propagating `L` around the unit-2
   orbit and inversion pairs;
8. the Lean-checked implication chain:

```text
unit/inversion equivariance
  => all six deletion rows have the required leading p-unit
leading p-unit nonzero
  => leading rank 156
  => leading erasure avoidance
  => delete-one right-orbit separation
  => right support >= 2
  => rank_Fp C_{157,211}=156.
```

Without the full-product-algebra equivariance data, the fallback fixed-factor
verifier should check the three opposite-prefix products and six tail products
directly.

The finite implication gates are:


`RepresentativeOnePUnitGate.lean` packages the whole finite chain in one

`TraceGcdOperatorRepresentativeGate.lean` packages the stronger trace-GCD

If the producer proves this operator norm detects any zero of the actual

row-goodness, unit-2 propagation, delete-one separation, and mixed rank.

## Missing Arithmetic Theorem

The missing theorem is no longer an existential minor search.  In the
equivariant full-product-algebra formulation it is:

```text
For the actual embedded p24 mixed Hermitian periods, the representative
leading Moore product L listed in the equivariant manifest is a p-unit.
```

Equivalently:

```text
the representative 156-coordinate Moore determinant is nonzero modulo p, and
the full cyclotomic product-algebra equivariance transports it to every
deletion row.
```

This is a finite-field/class-field p-unit theorem for named CM periods.  A
proof of it would give a strict certificate surface whose formal degree is
`m+n`, far below `sqrt(p)`.

Without the equivariance packaging, the fallback arithmetic theorem is the
direct nonvanishing of the three opposite-prefix products and six tail
products.

The current CS-theory boundary for this missing theorem is recorded in:

```text
p24/representative_cs_theory_candidate_boundary.md
p24/representative_kernel_cs_boundary_audit.py
```

It rules out a literal `16`-dimensional subfield explanation for the prefix
kernel and confirms by random controls that tail injectivity is not a formal
consequence of full prefix rank.  The remaining plausible import is therefore
rank-metric/subspace-polynomial language plus an arithmetic norm/resultant
p-unit proof for `L`.

## Current Boundary

The manifest and Lean gates are complete.  The actual p-unit values are not
constructed without enumerating the embedded class set.  Therefore this is a
certificate specification and proof target, not yet the final p24 certificate.
