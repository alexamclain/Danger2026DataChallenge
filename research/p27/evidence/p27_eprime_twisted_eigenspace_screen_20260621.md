# P27 E-Prime Twisted Eigenspace Screen

Date: 2026-06-21

## Claim

The explicit order-4 eigenspace object

```text
j^2 = -1
Z = T*(1 - j/X^3)
sigma(Z) = jZ
```

does not by itself expose `d3` or `d4` through the first visible Hilbert-90
packet:

```text
Z, Z^2, Z^4
Z +/- branch_factor
Z^2 +/- branch_factor
Z^4 +/- branch_factor
Z * branch_factor
base-field components of Z^4
```

over the non-degenerate guard fields `q=1471,1607,1847`.

This kills the nearest visible order-4 eigenspace screen.  It does not kill the
twisted/Prym route; it says the next test must compute the actual divisor or
Kummer class of the `d3`/`d4` double covers in the twisted model, not search
the obvious packet.

## Results

No exact packet appeared:

```text
q=1471: d3 exact_packets = 0, d4 exact_packets = 0
q=1607: d3 exact_packets = 0, d4 exact_packets = 0
q=1847: d3 exact_packets = 0, d4 exact_packets = 0
```

Best same-field scores:

```text
q=1471 d3 best = 272/400 = 0.680000000, packet = Z4-U2-1
q=1471 d4 best = 160/224 = 0.714285714, packet = Z4+U-1

q=1607 d3 best = 248/392 = 0.632653061, packet = Z4-U2+/-2U+4
q=1607 d4 best = 152/224 = 0.678571429, packet = Z2 / Z4 / base components

q=1847 d3 best = 360/504 = 0.714285714, packet = Z2 / Z4 / base components
q=1847 d4 best = 232/360 = 0.644444444, packet = Z4+U or Z4+U+4
```

The `Z2`, `Z4`, and some base-component winners are raw-majority artifacts:
they score well only when the target itself is biased in that guard field.
The nontrivial-looking packets do not persist across fields.

## Interpretation

Positive:

```text
The order-4 eigenspace model is now executable on the same d3/d4 quotient rows.
The tested packet is the natural first Hilbert-90 packet, not arbitrary fitting.
```

Negative:

```text
No named visible Z/Z2/Z4 packet gives an exact d3 or d4 source.
The best packets are field-dependent or raw-bias artifacts.
No GPU sampler follows from this visible eigenspace layer.
```
