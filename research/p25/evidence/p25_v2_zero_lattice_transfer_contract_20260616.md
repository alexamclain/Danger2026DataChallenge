## Lattice Data

Use edge order:

```text
(m1, m2, m4, m8)
```

The `W` boundary scale is the coefficient sum. The boundary-zero lattice has
rank `3`, generated by:

```text
m2 - m1 = (-1, 1, 0, 0)
m4 - m1 = (-1, 0, 1, 0)
m8 - m1 = (-1, 0, 0, 1)
```

All six pair quotients `m_j - m_i` are boundary-zero and span the same rank-3
lattice.

## Verdict

Boundary-zero quotient data is real and useful as transfer data. It can move a
known legal row theorem around the four-edge family, or normalize a nonunit
`W`-boundary theorem down to one edge if exact scalar-fixed zero-lattice values
are supplied. It is not a source-stage closer by itself: zero-boundary data has
no absolute `Norm_156(Y_507)` boundary and cannot create the first row value.

## Continue / Kill Recommendation

Continue to accept boundary-zero data only as a support/normalization route.
Kill any answer presenting row quotients, zero-lattice basis values, or
boundary-zero divisor relations as the first p25 source-stage theorem unless
it also supplies an absolute `W`-boundary row theorem or the exact zero-lattice
values needed to normalize one.
