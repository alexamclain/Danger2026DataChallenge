Since `75 = 3 * 25` with coprime factors, this source splits:

- kernel `C_25` is generated by `62^3 = 50`;
- visible `C_3` complement is represented by exponents `0,25,50`, values
  `(1,32,118)`;
- the formal `K` factor is a real split right-kernel trace, not a mystery

- kernel over visible `C_13` is `<354^13 = 246>`;

- every nonzero visible `C_13` class has only order-`169` lifts.

So the right side of `x^base * K_trace * D_segment * T_edge` can be understood
as a normal split trace plus visible right `C_3` data.  The hard arithmetic
piece is the C-axis: `D_c` and `T_c` require primitive cyclic `C_169` source
units, not independent order-`13` fiber units.
