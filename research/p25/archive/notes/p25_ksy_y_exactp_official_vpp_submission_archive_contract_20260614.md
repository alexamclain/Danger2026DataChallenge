# P25 KSY-y Exact-P Official VPP Submission Archive Contract

Updated: 2026-06-14 22:09 PDT

## Purpose

This contract starts after a concrete p25 `(p,A,x0)` exists and is claimed by
the exact-P / exact 75-atom route.  It extends the generic official `vpp.py`
archive contract rather than replacing it:

```text
concrete p25 triple
official vpp.py stdout True
generic archive bundle complete
Lean certificate generated and checked
exact-P source/bridge/surface/halving provenance archived
submission-ready state
```

An exact-P theorem path is not cleanly finished by an unarchived claim, a
failed verifier run, a generic archive with missing route provenance, or a
route packet without the official DANGER3 verifier boundary.

## Interpretation

For a future exact-P p25 payload, the immediate verifier command is:

```sh
python3 src/vpp.py 10000000000000000000000013 <A> <x0>
```

If and only if it prints `True`, preserve the generic official archive and then
attach the exact-P route provenance.  A production-search hit can bypass
theorem provenance only if that bypass is explicitly recorded; it still cannot
bypass official `vpp.py`, logs, hashes, environment, or Lean status.
