# Exact P

## Purpose

Track the headline 75-atom exact-product moonshot without letting it blur back
into broad source rereading.

## Current Claim

Exact `P` is still the biggest prize, but it is no longer the first thing to
ask from the literature. The current source verdict is:

```text
the 75 atoms are fixed payload factors, not 75 tries
KSY gives normalized-y atom vocabulary
Sprang gives D=2 additive/differential vocabulary
Kubert-Lang gives exponent/dependence vocabulary
none of them currently emits the exact finite product theorem
```

So exact-P remains alive as the heavier upstream route into the same finite
target now shared by H0 and conductor 39, not the first-pass front door.

The 75 atoms are fixed payload factors, not 75 independent tries. First-pass
reading budget should return to exact-P only when a source explicitly promises
the exact 75-atom product, its orientation, or the
`75 -> 300 -> 12 -> 312 -> 156` bridge.

The finite side now has a compact accepted theorem interface:

```text
C = (47, 28), D = (22, 3), primitive K = (57, 0), orientation
base = C - D = (25, 25)
T = -2*C + K = (38, 113)
```

A source theorem may also emit an accepted equivalent finite interface,
especially a period-156 theta2 or theta2-inverse divisor/additive payload.
Generic exponent balance or a value-only unit without period-156 branch
selection is not enough.

The exact-P-to-unified spine is now verified:

```text
compact exact-P interface
-> exact equal-weight normalized-y product
-> theta2 / theta2^-1 certificate path
-> Y_507 / Norm_156 bridge spine
-> unified H0/conductor-39 support-156 target
```

This implication is one-way. A unified H0/conductor-39 value/divisor theorem
does not by itself recover the exact 75-atom product.

The reverse boundary is now explicit: a unified support-156 theorem should
route to post-theorem extraction, not exact-P recovery, unless it also supplies
`C,D,K,orientation`, the equal-weight 75 atoms, an accepted period-156 theta2
payload, or an explicit reverse reconstruction theorem.

The exact-P finite-geometry rigidity page is now cockpit-tracked rather than
only lane-linked. Its lightweight validator confirms the disjoint 75-atom
support, equal theta2/theta2-inverse weights, row-polynomial normal form, and
the rejection of missing-atom, nonuniform-weight, and generic ray-class
escapes.

The exact-P minimal hook is now the preferred heavy-route handoff. A source
answer must emit the compact `C,D,K,orientation` packet, the exact
equal-weight 75-atom theorem, an accepted theta2/theta2-inverse payload, or an
explicit reverse reconstruction theorem. Normalized-y vocabulary, raw
Kubert-Lang balance, generic ray-class generation, finite payloads without
source, or unified-target theorems without exact-P selector data are repair or
reject rows.

The exact-P candidate sweep now consolidates the heavy-route near-misses.
KSY normalized-y vocabulary, finite 75-atom fixtures, compact packet data,
Kubert-Lang exponent and primitive-word screens, theta2 period-156 support,
Sprang D=2 support, value bridges, and unified-target theorems are all
scaffolding or repair rows unless they emit one of the accepted exact-P hooks:
compact `C,D,K,orientation` with one accepted raw branch, exact equal-weight
75-atom theorem, accepted theta2/theta2-inverse divisor-additive payload with
period-156 bridge, or explicit reverse reconstruction.

The exact-P/theta2 lookup-row status now compresses that into the active heavy
ask. Exact-P remains live through compact `C,D,K,orientation`, exact
equal-weight 75 atoms, exact theta2/theta2-inverse payload with period-156
bridge, theorem-legal KL primitive word or mixed `C_3 x C_169` selector,
Sprang sparse specialization, or explicit reverse reconstruction. Generic
normalized-y, KL, Sprang, theta, distribution, finite-packet, or unified-target
language remains support/repair unless it supplies one of those hooks.

The exact-P 75-anchor bridge filter now separates two different uses of
`75`. An exact row-labeled `R_m^75` theorem would normalize through the
common-scalar/unit-power intake because `gcd(75,p-1)=1`. The exact-P
equal-weight 75-atom product is not that row-power shortcut; it remains a
heavy upstream source candidate only with orientation and the verified
`75 -> 300 -> 12 -> 312 -> 156` bridge.

The exact-P spine payload separation now pins the actual finite support
ladder: `75 -> 300 -> 12 -> 312 -> 156`, ending in a `+78/-78` Hilbert-90
lift. This is the constructive bridge into the unified H0/conductor-39 target,
but it is not a plain 75-atom H90 product, not a row-power `R_m^75` theorem,

The exact-P closure-template replay boundary now keeps the old Lane B closure
